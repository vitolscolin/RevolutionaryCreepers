# AGENTS.md

## TrialTwin AI Working Agreement

- Keep the product hackathon-friendly and demo-first.
- Prefer simple, explainable implementations over clever abstractions.
- Preserve the modular layout under `backend/app`, `backend/scripts`, and `frontend/src`.
- Follow the documented run order in `README.md` before changing setup steps.
- Prioritize end-to-end functionality over isolated optimizations.
- Use synthetic data only.
- Keep comments in English and only where they add clarity.
- When adding features, make sure the backend response shape still supports the frontend demo flow.
- Do not turn this into a biologically realistic digital twin. It is a lightweight clinical trial operations simulator.
